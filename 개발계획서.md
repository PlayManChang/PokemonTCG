# PJCS 포켓몬 카드게임 용어집 — 개발계획서

## 1. 배경 및 목적
- 사용자의 아들이 일본 **PJCS(포켓몬 카드게임 일본 챔피언십)** 에 출전한다.
- 일본어를 모르는 상태에서 대회 현장에서 쓰이는 **포켓몬 카드게임 용어·인사말**을 빠르게 찾아볼 수 있어야 한다.
- 목표: **모바일에서 한 손으로, 네트워크 없이도** 필요한 표현을 즉시 찾는 웹사이트.

## 2. 대상 사용자
| 항목 | 내용 |
|------|------|
| 주 사용자 | 초·중등 연령 한국인 포켓몬 카드 플레이어 |
| 일본어 수준 | 모름 (한글 발음 표기 필수) |
| 사용 환경 | 일본 대회장, 스마트폰, 짧은 대전 사이 시간, 불안정한 네트워크 |
| 핵심 니즈 | 빠른 검색 · 정확한 발음 · 예의에 맞는 표현 |

## 3. 핵심 기능 (MVP)
1. **카테고리별 용어 보기** — 상황별(인사, 대전 진행, 게임 용어, 저지 호출 등)로 분류.
2. **빠른 검색** — 한국어/일본어/발음 어느 것으로도 즉시 필터링.
3. **발음 듣기(TTS)** — Web Speech API(`ja-JP`)로 일본어 음성 재생.
4. **한글 발음 표기** — 일본어를 못 읽어도 따라 말할 수 있게.
5. **즐겨찾기/중요 표시** — 자주 쓰는 표현 강조.
6. **오프라인 동작(PWA)** — Service Worker로 캐싱, 현장에서 네트워크 없이 사용.

### 제외(비범위)
- 회원가입/로그인, 서버 DB, 다국어(한·일 외), 카드 시세/덱 빌더 등은 범위 밖.

## 4. 콘텐츠 구조
각 용어 항목 데이터 스키마:
```
{
  "id": 고유 ID,
  "category": 카테고리 ID,
  "ja": 일본어 표기(한자 포함),
  "reading": 읽는 법(히라가나/가타카나),
  "ko_pron": 한글 발음,
  "ko": 한국어 뜻,
  "note": 사용 상황/설명,
  "important": 중요 표현 여부(boolean)
}
```

### 카테고리(상황 기준)
| ID | 이름 | 설명 |
|----|------|------|
| greetings | 인사·매너 | 대전 시작/종료 인사, 예의 표현 |
| match | 대전 진행 | 선후공, 턴, 확인 요청 등 진행 표현 |
| game | 게임 용어 | 필드/존, 덱, 패, 카드 조작 용어 |
| cards | 카드 종류 | 포켓몬/트레이너/에너지 등 |
| status | 상태이상·효과 | 독, 화상, 잠듦, 마비, 혼란 등 |
| phrases | 자주 쓰는 표현 | 대전 중 짧은 의사소통 |
| judge | 저지·문제상황 | 심판 호출, 규칙 확인, 문제 상황 |
| numbers | 숫자·시간 | 수량, 시간 관련 표현 |

## 5. 기술 스택
| 영역 | 선택 | 이유 |
|------|------|------|
| 프론트엔드 | 순수 HTML/CSS/Vanilla JS | 빌드 불필요, 어떤 무료 호스팅에도 그대로 배포 |
| 데이터 | `data/terms.json` | 콘텐츠 추가/수정이 쉬움 |
| 오프라인 | PWA + Service Worker | 현장 네트워크 불안정 대비 |
| 음성 | Web Speech API | 추가 의존성 없이 일본어 발음 재생 |
| 호스팅 | 무료 정적 호스팅(GitHub Pages / Cloudflare Pages / Netlify) | 비용 0 |

## 6. 화면 구성
- **상단**: 제목 + 검색창(고정).
- **카테고리 탭/칩**: 가로 스크롤로 상황 선택.
- **용어 카드 목록**: 일본어(큼) · 읽는 법 · 한글 발음 · 뜻 · 🔊버튼 · ⭐버튼.
- **하단**: 즐겨찾기 모아보기.

## 7. 역할 분담 (Agent)
| Agent | 역할 | 산출물 |
|-------|------|--------|
| **planner(기획자)** | 기능·콘텐츠 기획, 우선순위, 용어 정확성 기준, 사용자 시나리오 | 명세, 카테고리/용어 목록 |
| **developer(개발자)** | HTML/CSS/JS 구현, 데이터 작성, PWA, 배포 설정 | 동작하는 사이트 |
| **reviewer(리뷰어)** | 일본어 정확성, UX/접근성, 오프라인, 코드 품질 검수 | 검수 리포트 |

> 정의 파일: `.claude/agents/planner.md`, `developer.md`, `reviewer.md`

## 8. 개발 단계
1. **[기획]** 카테고리·용어 데이터 설계 (planner)
2. **[데이터]** `data/terms.json` 작성 (developer)
3. **[구현]** 화면/검색/TTS/PWA 구현 (developer)
4. **[검수]** 일본어·UX·코드 리뷰 및 수정 (reviewer → developer)
5. **[배포]** 무료 호스팅에 배포, 모바일에서 동작 확인
6. **[운영]** 대회 전 용어 추가/보완

## 9. 디렉토리 구조
```
PJCS/
├─ index.html              # 메인 화면
├─ css/style.css           # 스타일
├─ js/app.js               # 검색/렌더/TTS 로직
├─ data/
│   ├─ terms.json          # 앱이 읽는 최종 용어 데이터 (자동 생성)
│   └─ sources/            # 원본 자료 (여기서 terms.json 생성)
│       ├─ base.json       # 초기 핵심 용어
│       ├─ batch1~4.json   # 사용자 이미지 전사물
│       └─ README.md       # 자료 추가 방법
├─ manifest.webmanifest    # PWA 매니페스트
├─ sw.js                   # Service Worker(오프라인)
├─ icons/                  # 앱 아이콘 (svg + png 192/512)
├─ scripts/
│   ├─ merge-terms.js      # sources/*.json → terms.json 병합(멱등)
│   ├─ gen-icons.js        # PNG 아이콘 생성
│   └─ serve.js            # 로컬 테스트 서버
├─ 개발계획서.md
└─ .claude/agents/         # Agent 역할 정의

데이터 카테고리(12): 인사·매너 / 대전 진행 / 확인·질문 / 효과·용어 /
진화 / 에너지 / 카드·트레이너 / 포켓몬체크·상태 / 사이드 / 주사위·코인 / 저지 / 숫자·시간
```

## 9-1. 자료 추가 워크플로우
1. 새 자료(사진/캡처)를 `Data/` 폴더에 넣는다.
2. Claude가 이미지를 읽어 `data/sources/batchN.json` 으로 전사한다.
3. `node scripts/merge-terms.js` 실행 → `data/terms.json` 재생성(중복 제거·멱등).
4. 데이터 수정은 반드시 `data/sources/` 의 소스 파일에서 (terms.json 직접 수정 금지, 덮어써짐).
   자세한 내용은 `data/sources/README.md` 참고.
